Evident Point

UI and front end developer
Evident Point specializes in customized digital publishing for clients. During the co-op placement, I was responsible for developing UI for Active Textbook, which is a digital tool that let user annotate PDFs and e-books with various media.
  • TASKS Research, conceptualization, creating mockups, page styling, identifying and fixing front end bugs
  • TOOLS HTML, SCSS, Javascript, Photoshop
  • CO-OP
  • FALL 2016 4 Month

Evident Point Website Active Textbook

Custom Feature UI

Other than customizing the Active Textbook portal color scheme to fit client’s requirements in via HTML and SCSS, some clients would request new custom features for their portal version.

For example, when I had to create a report filter feature, I had to create mock-up implementations and consult often with the back-end developers. During this process, other considerations were spent on scalability in terms of report amount, screens size, and information hierarchy.

Bugs & UI Enhancement

Other than custom features, I would receive Jira tickets to create UI enhancements of certain pages in the portals. But most of the time, I would be working on fixing UI bugs that have appeared after deployments. In these cases, I would check out the work branch to find the bugs and then meet with the lead programmer of the deployment to give them a heads up or for further steps.

Icons

Clients with different needs usually require new sets of UI icons to be made based on their color scheme or new portal features. In most cases, researching for an icon that can be easily decipherable when scaled into a small size is the first step. If not, creation and adjustments will be made in Illustrator and then transferred into the icon sheet via Photoshop.

Documentation

Throughout my placement, I had to navigate through the large codebase of the Active Textbook portal. Initially, it was daunting and confusing with many unused files and an outdated guide which made the onboarding period even longer and more frustrating. To tackle this problem, I worked with a senior developer on cleaning up the files and revamping the file structure, so it would match user’s mental map easier. By updating the documentation, the next co-op student wouldn’t have to spend as much time navigating the code base and other developers can also use the guide to make portal customization changes if need to.
